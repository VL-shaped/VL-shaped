Domain Name Service - DNS


DNS Caching

Operating systems maintain a temporary database called a DNS cache that contains a historical record of all viewed and searched web pages and websites. Before a browser sends a request to the internet, the OS intercepts each one and looks up the domain name in the DNS cache database, if it is there it is loaded much quicker and easier than accessing the router and DNS Server.

The Domain Name System

A domain is an area, your domain is the area you occupy. Networks use the same word to describe an area inside a network. A domain belongs to someone and can only be accessed if you are granted authorisation by the owner. The Internet uses domains and domain names to describe where a website’s files are hosted (stored on a server). Example domain name: www.bbc.co.uk. The Domain Name System is a set of servers called Nameservers that store IP addresses of devices connected to the Internet, in a database. Every domain name is connected to a specific IP address. DNS is like an address book; it converts domain names into IP addresses so that web browsers know where the web pages are located (known as DNS resolution). All computers/servers have an IP address for example: 192.168.2.10. Domain names are used as they are more memorable to humans than a numerical address.

DNS Servers

The DNS service uses four specific types of DNS Server to run DNS queries and DNS Lookup and Resolution:

1. DNS Recursor – First stage in DNS query, also known as a recursive resolver. Receives request from client then either responds with cached data or sends request to the next servers in the process to acquire the data.

2. Root Nameserver, Root Server – The root server accepts the query (including a domain name) and responds by directing it to the relevant TLD nameserver (based on domain extension).

3. TLD Nameserver – There are many TLD Nameservers each containing all the information on a particular domain extension (for example .com). A request is then sent to the required authoritative nameserver.

4. Authoritative Nameserver – Holds the information on the entire domain name (e.g. google.com) including its IP address. This is then sent back to the recursive resolver with the destination IP address of the websites files to be displayed to the client.
